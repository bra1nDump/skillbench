- **NEW CONTENDER ALERT — Emdash orchestrator** (Hacker News https://news.ycombinator.com/item?id=47140322, Show HN post). The open-source Emdash desktop app now runs 21 coding agent CLIs in parallel (Claude Code, Codex, Gemini, Droid, Amp, Codebuff, etc.), isolates each task in worktrees, and push-notifies you when a remote agent is ready. Why this matters: it turns multi-agent sessions into a workflow hub that already supports the exact CLI roster we track, so it could be a valuable catalog entry rather than just another wrapper. Source freshness: 2026-02-24, 206 points + GitHub repo https://github.com/generalaction/emdash.
- **New multi-worktree multitasking stack — Superset** (Hacker News https://news.ycombinator.com/item?id=46109015). Superset uses one-click git worktrees per agent, isolates terminals, and spotlights agent-specific notifications so teams can run Codex, Claude, Gemini, etc., simultaneously without losing context. Why this matters: early adopters report 2–3× faster developer velocity, making it a practical orchestration layer the catalog lacks; it pairs naturally with our CLI ranking tiering. Source freshness: 2025-12 run with 24 points and https://github.com/superset-sh/superset.
- **Shared memory for agents — Grov** (Hacker News https://news.ycombinator.com/item?id=46711958, https://grov.dev/). Grov logs decision-grain memories, branches them like git, and injects summaries only when the active agent asks (preview→expand), cutting token waste by ~50–70% while preserving architectural intent across Claude Code/Gemini/Codex sessions. Why this matters: it addresses persistent-context gaps our catalog doesn’t cover (no memory layer), and the approach is already being dogfooded across multi-agent teams. Source freshness: 2026-01-21, 24 points, GitHub https://github.com/TonyStef/Grov and live site.
